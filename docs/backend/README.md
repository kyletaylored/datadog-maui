## Recent Fixes

### RUM Configuration Loading
**Issue**: Browser RUM wasn't loading because RumConfigController was reading from Web.config appSettings that don't exist.

**Fix Applied**: Updated [RumConfigController.cs](../../ApiFramework/Controllers/RumConfigController.cs) to serve the static rum-config.js file generated by the PowerShell script during build, instead of generating it dynamically from Web.config.

**To test on Windows**:
1. Open the project in Visual Studio on Windows
2. Build the project (this runs generate-rum-config.ps1)
3. Verify rum-config.js is generated in ApiFramework/ directory
4. Run the application and navigate to http://localhost:50000
5. Open browser console and verify you see: `[Datadog] RUM config loaded from .env file`

### APM Tracing in IIS Express
**Issue**: Environment variables for Datadog CLR profiler not being picked up by IIS Express process.

**Status**: Configuration script created ([configure-iis-express.ps1](../../ApiFramework/configure-iis-express.ps1)) but needs Windows testing.

**To fix on Windows**:
1. Install [Datadog .NET Tracer MSI](https://github.com/DataDog/dd-trace-dotnet/releases) if not already installed
2. Run `.\configure-iis-express.ps1` from PowerShell in the ApiFramework directory
3. Close Visual Studio completely and reopen
4. Build and run the project
5. Verify with: `dd-dotnet check process <pid>` (find PID from Task Manager)
6. If variables still not set, see [IIS Express Datadog Setup](IIS_EXPRESS_DATADOG_SETUP.md) for alternative approaches
